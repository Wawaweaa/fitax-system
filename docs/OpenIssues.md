# 开放问题与后续关注

1. **鉴权/租户**：当前假设 `user_id` 已从认证层注入，实际方案（JWT/Cognito/自建）待定。
2. **Storage driver**：AliOSS/MNS/S3 具体 SDK 实现与测试需确认；是否引入统一抽象库。
3. **merge/replace 策略**：`row_key` 推导细节（特别是缺 `line_no` 的平台）需进一步验证样本。
4. **金样维护**：expected CSV 更新流程与版本控制方式待制定（避免误提交差异）。
5. **日志与监控**：接入平台（Sentry/Datadog + 阿里云 SLS）和告警阈值待确认。
6. **部署自动化**：staging/prod CI/CD 流程、脚本尚需完善（terraform/脚本）。
7. **复式导出**：current 仅支持按月导出；是否需要按平台汇总导出多月、合并 Zip。P1 后评估。
8. **append 模式**：留待 P1 设计；与 merge/replace 的共存策略尚未落地。
9. **前端样式精修**：留白、字体、色彩等细节待在功能稳定后统一处理。
10. **大文件 & 并发**：需实测 50MB 文件与多任务并行的性能；若有瓶颈考虑 chunk 上传、并发控制。
11. **微信视频号字段适配**：当前已适配基本字段，但可能存在更多非标准命名变种；需收集更多样本进行测试和完善适配。
12. **聚合计算精度**：微信视频号数据存在小数金额，聚合计算中可能产生0.01-0.02的小数误差；需考虑是否增强精度处理。
13. **预置映射模板**：考虑为常见文件格式提供预置映射模板，减少手动映射需求，尤其是微信视频号文件格式存在变化的情况。
14. **DuckDB性能优化**：随着数据量增长，需评估DuckDB查询性能，考虑增加索引或分区策略。
